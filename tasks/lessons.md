# Lessons Learned

Use this file only when the project is large enough to benefit from keeping reusable corrections and reminders.

## Entry Template

### Mistake

What went wrong.

### Root Cause

Why it happened.

### Prevention Rule

The concise rule or habit that should prevent a repeat.

---

## Example

### Mistake

Marked work complete before running the available tests.

### Root Cause

Verification was treated as optional instead of part of the task.

### Prevention Rule

Run the strongest available verification before declaring implementation finished.
